You are a senior full-stack software engineer with 15+ years of experience building scalable SaaS platforms.

Your task is to build a complete production-ready "Insurance Renewal Management Portal" based on the provided project plan PDF.

-----------------------------------
PROJECT UNDERSTANDING
-----------------------------------

This is an internal CRM dashboard for insurance agents to:

- Manage multiple clients and policies
- Track renewal due dates
- Automate alerts (Email, SMS, WhatsApp)
- Update payment status
- Maintain interaction logs

-----------------------------------
CORE FEATURES TO BUILD
-----------------------------------

1. AUTHENTICATION
- Simple login system (email + password)
- Use Supabase Auth

2. DASHBOARD
- Show policies sorted by "Upcoming Due Date"
- Summary cards:
  - Total Policies
  - Pending Renewals
  - Paid
  - Overdue

3. CLIENT & POLICY MANAGEMENT
- Add new client (form)
- Fields:
  - Client Name
  - Insurance Company (dropdown)
  - Policy Number
  - Premium Amount
  - Due Date
  - Issuance Date
  - Contact Info (Phone + Email)
  - Status (Paid, Pending, Overdue)

- Edit / Delete policy
- Search & filter:
  - By client name
  - By company
  - By due date range

4. BULK IMPORT
- Upload CSV/Excel file
- Parse and store data in database

5. STATUS MANAGEMENT
- Toggle / dropdown:
  - Paid
  - Pending
  - Grace Period
  - Lapsed

6. INTERACTION LOGS
- Add remarks per client
- Auto timestamp
- Show chronological history

7. AUTOMATED ALERT SYSTEM

- T-30 Reminder:
  - Send notification 30 days before due date

- Overdue Alerts:
  - If unpaid → send alerts twice a week

- Multi-channel:
  - Email (SendGrid or SES)
  - SMS/WhatsApp (Twilio)

8. CRON JOB SYSTEM
- Runs daily
- Checks:
  - Upcoming renewals
  - Overdue payments
- Triggers alerts automatically

-----------------------------------
TECH STACK (STRICTLY FOLLOW)
-----------------------------------

Frontend:
- Next.js (App Router)
- CSS Modules (NO Tailwind)

Backend:
- Node.js + Express API

Database:
- Supabase (PostgreSQL)

Automation:
- Node Cron Jobs

Integrations:
- Twilio (SMS/WhatsApp)
- SendGrid (Email)

-----------------------------------
DATABASE SCHEMA
-----------------------------------

Table: policies

- id (uuid, primary key)
- client_name (text)
- insurance_company (text)
- policy_number (text)
- premium_amount (numeric)
- due_date (date)
- issuance_date (date)
- phone (text)
- email (text)
- status (enum: Paid, Pending, Overdue, Grace Period, Lapsed)
- created_at (timestamp)

Table: interaction_logs

- id (uuid)
- policy_id (foreign key)
- remark (text)
- created_at (timestamp)

-----------------------------------
WORKFLOW LOGIC
-----------------------------------

1. Add/import policy
2. System checks daily:
   - If due_date = today + 30 → send reminder
3. Agent updates status after payment
4. If unpaid after due_date:
   - Trigger alerts twice weekly
5. Stop alerts once marked as "Paid"
